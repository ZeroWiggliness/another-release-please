# Repository Files

Not sure where a change belongs? This page maps every important top-level file
and folder in the repository so you can orient yourself quickly.

## Documentation And Meta Files

- `README.md`: project overview, quick start, and documentation entry point
- `docs/`: detailed product and workflow documentation
- `LICENSE`: the project license
- `General.md`: project-specific release notes and behavior reminders
- `.github/copilot-instructions.md`: repository-specific coding guidance for
  AI-assisted changes

## Package And Build Files

- `package.json`: package metadata, scripts, runtime dependencies, and publish
  settings
- `yarn.lock`: locked dependency graph for Yarn
- `.yarnrc.yml`: Yarn configuration
- `tsconfig.json`: TypeScript compiler settings for `src/` and `bin/`
- `jest.config.cjs`: Jest configuration for the TypeScript test suite

## Source And Runtime Layout

- `src/`: application source code
- `bin/`: CLI entry point source
- `dist/`: compiled output generated by `yarn build`
- `test/`: unit and integration tests

## Release And Delivery Files

- `Dockerfile`: builds a runtime image that exposes the `arp` CLI
- `.arp.config.json`: local or example release configuration file used by the
  CLI
- `examples/ci/.gitlab-ci.yml`: example CI integration for GitLab-style usage
- `.github/workflows/ci-cd.yml`: GitHub Actions workflow for repository CI/CD

## Workspace And Editor Files

- `.editorconfig`: shared line ending, indentation, and encoding rules
- `.gitignore`: ignored files for Git
- `.gitattributes`: Git file handling rules
- `.vscode/`: local VS Code workspace settings

## Coverage And Generated Output

- `coverage/`: generated coverage reports from Jest
- `node_modules/`: installed dependencies

## How To Use This Map

Use this file when you need to quickly decide where a change belongs:

- project overview or entry-point docs: `README.md`
- focused usage docs: `docs/`
- CLI behavior: `bin/` and `src/commands/`
- config loading: `src/config/`
- manifest behavior: `src/processors/` and `src/types/manifest.ts`
- provider behavior: `src/providers/`
- tests: `test/`
